§1.1: Introduction
  • OS sits between the hardware and software of the system so that the hardware is effectively used by the apps (Hardware Abstraction), and manage fairness & security (Resource Management).
    • Hardware Abstraction: Suppose you want to print “Hello World”; without an OS you will have to specify the colour, coordinates, program the graphics card i.e. handle every single nitty gritty. The OS sits in the middle & provides an easy reusable function that every app can use.
    • Resource Management: OS ensures that the limited hardware is shared fairly across different applications, ensures security & isolation between apps, and improve performance.
  • There are various kinds of OS’s for different use cases. Real Time Operating Systems (RTOS) provide guarantees wrt time and is used in critical infrastructure like autonomous cars, rockets, etc.
§1.2: OS Basics
  • Multiprogramming uses a queue structure and waits for the first application to terminate before starting another. If the first program malfunctions and doesn’t terminate, then the 2nd app would never start. Multiprogramming is prone to hanging / starvation
  • Modern OS’s support Multitasking (Time Share): Entire time in the OS is sliced and each slice is given to an app. This gives the impression of concurrency & improves performance (NO Starvation)
  • Multi-processors, cores, and threads enable true concurrency & the OS should be able to handle race conditions
    • The way modern OS’s handle race condition is by using something called Synchronisation. The shared resource is associated with a lock which ensures that only one app can access it at a time.
  • Scheduling: OS determines which apps to execute next and handle priority queues.
  • Isolation: Should isolate applications from one another and also to isolate OS from the applications (A malfunctioning app should not cause the OS itself to malfunction)
    • Intel uses rings: 0 (Most Privileged - OS/Kernel), 1,2, 3 (Least Privileged - user apps). Any application running in ring 3 can not access any data or information present in in ring 2,1,0
    • RISC-V uses three modes:
      • Machine Mode: Instructions executing have full privilege; a CPU starts in machine mode. Machine mode is mostly intended for configuring a computer. Xv6 executes a few lines in machine mode and then changes to supervisor mode.
      • Supervisor Mode: In supervisor mode the CPU is allowed to execute privileged instructions: for example, enabling and disabling interrupts, reading and writing the register that holds the address of a page table, etc. If an application in user mode attempts to execute a privileged instruction, then the CPU doesn’t execute the instruction, but switches to supervisor mode so that supervisor-mode code can terminate the application, because it did something it shouldn’t be doing.
      • User Mode: User Apps are run here. Each app is given virtual memory & sandboxed. Each app can also have different page permissions (rwx).
  • Security: Defend against internal or external attacks from viruses, etc. This is achieved using Access Controls, Passwords, Biometrics & Security Assessments.
§1.3: OS Concepts & Structure
  • Code creates an executable file. Running this executable file creates a Process. A Process is a program in execution & comprises of executable instructions, stack, heap, state (registers, open files, related processes). An OS typically holds the state of the process.
    • Processes are run in user mode and they have restricted access to resources.
    • The OS runs in Kernel Mode
  • System Call invokes a function in the kernel using a Trap which causes the processor to shift from the user mode to kernel mode and on completion gets transferred back to user model. They should offer sophisticated features, be simple and abstract whatever is necessary

    System Call

    - Uses Trap Instruction (ECALL)

    - System Shifts from user to kernel mode

    - TRAP always jumps to fixed address

    Procedure Call (func1 calling func2)

    - Uses CALL instruction

    - Stays in user space

    - Relocatable address

    printf invokes the system call write(STDOUT)

    • In RISC-V, ECALL transfers from user mode to machine mode, which delegates to supervisor mode and then returns after the system call is complete. It stores the current mode (user, supervisor, machine) in register mstatus and the program counter in EPC (Exception Program Counter).
  • An OS will generally have many modules like System Call Interface, Memory Management, Networking Stack, Filesystem Management, CPU Scheduling, Inter Process Communication, Device Drivers, etc

    Monolithic Structure

    - All components of the OS lie in the kernel space and can easily invoke each other using a simple function call ⇒ Easy Communication

    - Large Size, difficult to maintain

    - Used in Linux, MS-DOS, xv6

    Microkernel Structure

    - Highly Modular: Every component has its own space and interaction between components goes through well defined interfaces. Most of the components will lie in the user space

    - Kernel space will have basic inter-process communication and scheduling

    - Used in QNX, L4

§1.4: System Organization
  • The two important pulses in a system are i) Reset and ii) Clock. The reset signal which tells the CPU to restart and to go into some initialisation mode. The clock is used by the CPU and and other transistors to perform various operations.
  • The system bus is a set of connections that come out of the CPU and are used to connect various modules to the processor.
  • There are multiple types of memory i) Volatile / DRAM (Program Data, Stack, Heap lie here), ii) Read Only Memory (Typically has the initial portions of the program that first get executed).
  • A Keyboard connected to the USB controller would send a signal to the interrupt controller so that the keystrokes can be read.
  • One critical aspect in order to communicate to each of these components are the two different types of addresses
    1. Memory Addresses: Present in the RAM or the ROM
    1. Memory Mapped IO Addresses: Memory maps are range of addresses with each range associated with a component from the system bus (RAM, Display, Mouse, Keyboard). Each Controller will have limited registers associated with the addresses that control the functionality.
